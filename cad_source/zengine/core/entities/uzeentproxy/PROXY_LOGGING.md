# Логирование ProxyEntity

Загрузка и разбор `ACAD_PROXY_ENTITY` используют отдельный модуль
`programlog` с именем `PROXY`. Модуль регистрируется в
`uzeentproxylog.pas` без `EEnable`, поэтому по умолчанию подробная
диагностика ProxyEntity выключена и не попадает в обычный лог загрузки.

## Как включить

Для записи ProxyEntity-диагностики включите файл лога и модуль `PROXY`:

```text
zcad ... logfile <path> lem PROXY
```

После этого сообщения из парсера Proxy Graphic, диспетчера OpCode,
построителей подпримитивов и `GDBObjAcdProxy` будут попадать в указанный
лог. Обычные сообщения других модулей при этом не требуется включать.

Если запуск уже включает все модули через `leam`, то `PROXY` тоже будет
включен. Чтобы оставить общий лог, но убрать подробную ProxyEntity
диагностику, отключите только этот модуль:

```text
zcad ... ldm PROXY
```

## Как выключить

Самый простой способ выключить диагностику ProxyEntity - не передавать
`lem PROXY` в параметрах запуска. В релизном режиме это поведение по
умолчанию.

Если модуль уже был включен явно или через `leam`, отключите его командой:

```text
zcad ... ldm PROXY
```

Для чистого обычного лога запускайте ZCAD без `lem PROXY` и без общего
`leam`, либо добавляйте `ldm PROXY` после включения всех модулей.

## Как это устроено

Модуль `uzeentproxylog.pas` регистрирует собственный `ModuleDesk`:

```pascal
ProxyLogModuleId := programlog.RegisterModule(PROXY_LOG_MODULE_NAME);
```

Так как регистрация выполняется без `EEnable`, сообщения не выводятся,
пока пользователь не включит модуль средствами `programlog`.

Внутренний код ProxyEntity пишет диагностику только через хелперы:

- `ProxyLogInfoFormatStr`
- `ProxyLogWarningFormatStr`
- `ProxyLogErrorFormatStr`

Новые диагностические сообщения для загрузки ProxyEntity нужно добавлять
через эти хелперы, а не через `zDebugLn`, `DebugLn` или прямой
`programlog.LogOutFormatStr`.
